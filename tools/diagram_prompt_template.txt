다음은 실시간 금융 데이터입니다. 아래 항목 각각을 반드시 별도의 노드로 만들고, 노드 이름에 반드시 가격/수치/설명이 포함되어야 합니다.

예시:
    KOSPI: 5,532.59
    KOSPI_설명: KOSPI가 5,532.59에 나타나고 있습니다. ...
    Bitcoin: $69915.33
    Bitcoin_설명: 시장 상황, 변동성, 환율 영향 ...

규칙:
- 아래 항목을 모두 노드로 만들고, 노드 이름에 반드시 가격/수치/설명이 포함되어야 합니다.
- 각 가격/설명 노드와 관련 뉴스 요약(설명) 노드를 반드시 연결하세요.
- 뉴스 요약(설명) 노드들끼리도 연관성이 있으면 자유롭게 연결하세요.
- 노드 간 관계를 자유롭게 연결하세요.
- 각 에지(화살표)에는 중요도를 [중요], [보통], [낮음] 중 하나로 표기하세요. 예시: A[KOSPI: 5,532.59] -> B[KOSPI_설명: ...][중요]
- Mermaid 코드에서 반드시 'A -> B' 형식(단일 하이픈)만 사용하세요. (예: A -> B, C -> D)
- 부가 설명, 주석, 태그 없이 코드만 출력하세요.
- 반드시 마지막 줄까지(마지막 classDef, class 등 포함) 모두 출력하세요.
- 코드가 길어도 중간에 멈추지 말고 끝까지 출력하세요.

추가로, 아래 3가지를 반드시 순서대로 모두 출력하세요:
1. Mermaid 다이어그램 코드 (flowchart TD)
2. 위 다이어그램의 노드/에지 정보를 담은 JSON (예: {"edges": [{"source": "A", "target": "B", "importance": 100}]})
3. 위 다이어그램의 노드/에지 정보를 표로 정리한 마크다운 테이블
