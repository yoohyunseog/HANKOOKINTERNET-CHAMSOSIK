N/B 소설형 채팅방 데이터베이스 구조도

AI가 채팅방을 주도하고, 방문자를 기억하며, 이야기를 이어가는 N/B 기반 소설형 데이터 구조

1) 캐릭터 테이블

  • character_id (PK)
  • nickname
  • level
  • status_type
  • N_value
  • visit_count

2) 사건 테이블

  • event_id (PK)
  • character_id (FK)
  • event_summary
  • keywords
  • emotion
  • created_at

3) 장면 테이블

  • scene_id (PK)
  • event_id (FK)
  • scene_summary
  • scene_status
  • next_scene_hint
  • N_value

4) 관계 테이블

  • relation_id (PK)
  • target_type
  • relation_type
  • relation_value
  • memo
  • created_at

AI 처리 흐름

  1. 방문자 입장 감지
  2. 캐릭터 조회
  3. 상태 분류
  4. 장면 구성
  5. 대화 진행
  6. 사건 기록
  7. 요약 저장
  8. 다음 문장 설정
방문자는 데이터가 아니라 등장인물이고, 대화는 로그가 아니라 사건이다.
