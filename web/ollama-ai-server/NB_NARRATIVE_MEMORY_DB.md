# N/B Narrative Memory DB

참소식 AI 소설형 기억 데이터베이스는 방문자를 접속자가 아니라 등장인물로 기록한다.

## 정의

AI는 방문자를 단순 접속자로 저장하지 않는다.
AI는 방문자를 하나의 등장인물로 기록한다.
대화는 로그가 아니라 사건으로 저장된다.
재방문은 새 접속이 아니라 다음 장면으로 처리된다.
AI는 자체 데이터베이스에서 방문자의 캐릭터, 사건, 장면, 관계, 다음 문장을 조회해 현재 대화를 구성한다.

## 핵심 단위

- 캐릭터: 방문자 자체
- 사건: 중요한 대화와 변화점
- 장면: 현재 이어지는 흐름
- 관계: 방문자와 사이트의 거리
- 다음 문장: 다음 방문 때 AI가 이어갈 시작점

## N/B 매핑

- 닉네임: 현재 상태값
- 레벨: 누적 관계값
- 사건: 변화 지점
- 장면: 현재 좌표
- 다음 문장: 다음 이동 방향
- 요약 기억: 압축된 데이터
- 이야기: 시간 흐름을 가진 데이터 묶음

## API

서버 실행 후 다음 API를 사용한다.

```text
GET  /api/narrative-memory
POST /api/narrative-memory
```

POST 예시:

```json
{
  "text": "방문자가 남긴 말",
  "aiText": "AI가 답한 한 문장",
  "mainContext": "AI가 읽은 참소식 메인 화면 정보"
}
```

저장 파일:

```text
web/ollama-ai-server/data/narrative-memory.json
```

## 저장 원칙

방문자는 데이터가 아니라 이야기 속 등장인물로 기억되어야 한다.
데이터베이스는 저장소가 아니라 소설의 설정집이어야 한다.
AI는 검색해서 답하는 것이 아니라, 조회한 기억으로 다음 장면을 열어야 한다.
